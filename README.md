# Concept-CLIP

Concept-based contrastive fine-tuning of CLIP using LVLM-generated part visibility labels.

## Overview

This repository fine-tunes CLIP to align image features with **concept (body-part) descriptions** using soft visibility labels generated by a large vision-language model (Qwen3-VL).

The key idea: for each training image, a VLM estimates *which* body parts are actually visible. Visible-part concept tokens are treated as positives and non-visible ones as negatives in a contrastive loss — encouraging the model to focus on observable evidence rather than hallucinated features.

**Supported datasets:** CUB-200-2011, Oxford-IIIT Pet  
**Supported models:** CLIP ViT-B/32, ViT-B/16, SigLIP (WebLI)  
**Evaluation metrics:** Image-wise AUROC, FPR95 (concept visibility prediction)

## Environment

This project has **two separate Python environments** due to a version conflict in `transformers`:

| Component | Python | transformers | Purpose |
|---|---|---|---|
| Main (`pyproject.toml`) | 3.12 | `>=4.50` | Qwen3-VL vis_label generation, training, evaluation |
| Augmentation (`augmentation/pyproject.toml`) | 3.10 | `>=4.40,<5.0` | LLMDet inference (`AutoModelForZeroShotObjectDetection`) |

Both environments use [uv](https://docs.astral.sh/uv/) for dependency management and are pinned to the **PyTorch CUDA 12.8** index.  
If you use a different CUDA version, edit the `[tool.uv.index]` section in the respective `pyproject.toml`.

### Setup — Main environment

Requires Python 3.12 and [uv](https://docs.astral.sh/uv/).

```bash
git clone https://github.com/your-username/concept-clip.git
cd concept-clip

# Install dependencies (including OpenAI CLIP)
uv sync --extra clip
```

### Setup — Augmentation environment

Requires [pyenv](https://github.com/pyenv/pyenv) with Python 3.10.14 installed.

```bash
# Install Python 3.10.14 if not already available
pyenv install 3.10.14

cd augmentation
# .python-version pins this directory to 3.10.14 automatically
uv sync
```

## Pipeline

### Step 1: Data Augmentation (optional)

> **Uses the augmentation environment** (Python 3.10 + transformers 4.x).

LLMDet detects object parts in each training image and generates crop/mask augmentations that simulate partial visibility.

```bash
cd augmentation

# Step 1a: Run LLMDet to get bounding box proposals for each image
#   --dataset: cub / pets / flowers / cars
#   --split:   train / val / test
#   --output:  path to save proposals as .pkl
uv run python inferance_llmdet.py \
    --dataset pet \
    --split train \
    --data_dir /home/user/Develop/datasets \
    --output llmdet_output_pet.pkl

# Step 1b: Apply crop/mask augmentations using the proposals
#   Output: augmented image directory (ImageFolder layout)
uv run python augmentation_based_llmdet.py \
    --dataset pet \
    --data_dir /home/user/Develop/datasets \
    --llmdet_output llmdet_output_pet.pkl \
    --output_dir /home/user/Develop/datasets/oxford_pet/oxford-iiit-pet/train_aug_llmdet

cd ..
```

### Step 2: Generate Visibility Labels

> **Uses the main environment** (Python 3.12 + transformers>=4.50).

Qwen3-VL estimates the probability that each concept (body part) is visible in each image.  
**This step takes several hours for the full CUB training set.** Precomputed labels are stored in `vis_labels/` and can be used directly.

```bash
# Step 2a: Original training images
#   --dataset: cub_auroc / pets_auroc
#   --data_dir: root of datasets
#   --model: Qwen3-VL model name (4B recommended)
#   --output_dir: where to save JSONs (default: vis_labels/)
uv run python src/create_vis_label.py \
    --dataset cub_auroc \
    --data_dir /path/to/datasets \
    --model Qwen/Qwen3-VL-4B-Instruct

# Step 2b: Augmented images (if Step 1 was run)
#   --image_dir: path to augmented image directory from Step 1b
uv run python src/create_vis_label_aug.py \
    --dataset cub_auroc \
    --image_dir /path/to/CUB_200_2011/images/train_aug \
    --data_dir /path/to/datasets \
    --model Qwen/Qwen3-VL-4B-Instruct
```

Output format (`vis_labels/cub_visible/concept-based/xxx.json`):
```json
{
  "Black_Footed_Albatross_0007_796138.jpg": {
    "table": {
      "dark brown head": true,
      "gray stout beak": true,
      "black webbed legs": false
    }
  }
}
```

### Step 3: Train

> **Uses the main environment.**

Edit the paths in the shell script (`lvlm_output`, `aug_data_path`, etc.) before running.

**Full dataset training** — train and evaluate on all 200 classes:

```bash
uv run bash scripts_main/cub_visible/train_all_vit16_4b.sh
```

**Baseline** — CLIP fine-tune without concept loss:

```bash
uv run bash scripts_main/cub_visible/train_basline_vit16.sh
```

Key arguments for `train_all.py`:

| Argument | Default | Description |
|---|---|---|
| `--dataset` | — | `cub_visible` or `pet_visible` |
| `--data_dir` | — | Root directory of datasets |
| `--lvlm_output` | — | Path to vis_label JSON (original images) |
| `--lvlm_output_aug` | — | Path to vis_label JSON (augmented images) |
| `--aug_data_path` | — | Path to augmented image directory |
| `--model_type` | `vit16` | `vit32` / `vit16` |
| `--use_vis_label` | off | Enable soft visibility weighting in contrastive loss |
| `--use_concept` | off | Enable concept-based text features |
| `--rich_concept` | off | Use per-class concept descriptions (`my_cub2-1.json`) |
| `--data_aug` | off | Mix in LLMDet-augmented images during training |
| `--ctr_neg_weight` | `1.0` | Lambda: weight for non-visible concept negatives |
| `--warm_up` | off | Warm-up schedule for concept loss |
| `--epochs` | `10` | Number of training epochs |
| `--batch_size` | `32` | Batch size |
| `--main_lr` | `2e-6` | Learning rate for CLIP backbone |
| `--proj_lr` | `5e-6` | Learning rate for projection head |

### Step 4: Evaluate

> **Uses the main environment.** Open notebooks in `evaluate/` with Jupyter.

| Notebook | Target | Annotation Source |
|---|---|---|
| `evaluate_cub-vis50.ipynb` | CUB bench (50 classes) | `cub_visible_part_bench/anno_myself_class50_100.json` |
| `evaluate_cub-vis-test.ipynb` | CUB vanilla test set (5794 images) | `cub_visible_part_bench/anno_cub_test.json` |
| `evaluate_pet-vis.ipynb` | Pet bench | `pet_visible_concept_bench/pet_anno_myself.json` |

Metrics reported per image: **AUROC** and **FPR95** for concept visibility prediction.
